Document compilation, vault maintenance (background)

! Clerk runs continuously in the background

The clerk calls an LLM for every document in your workspace and keeps watching for new files. With cloud providers this can consume tokens rapidly — especially the first time it ingests a large workspace.

Recommended: use a local model (LM Studio, Ollama) or a cheap/fast one (Claude Haiku, GPT-4o mini). Avoid pointing this at Opus-tier models unless you're comfortable with the spend.
